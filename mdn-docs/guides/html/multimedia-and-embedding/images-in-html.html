Images in HTML

Web without images are really boring. Here we are going to learn how to use images in HTML

How do we put an image on a webpage?

To put image in web page we use <img> tag. The <img> tag is a void element, which means it doesn't have any children and end tag. We use src attribute to specify the source of the image.

Search engines also counts filename towards SEO. Therefore we should give proper filename to the image, html.jpg is better name than img123.jpg

Pointing the src attribute to an image in someone else hosted website is hotlinking. It is considered a unehtical because someone else is paying for bandwidth.

Alternative text

It is the textual description of the image. If the image is not able to load.

The reason we need a alt text are

  • It is used to describe the screen reader.
  • Spelling of the filename or path can be wrong.
  • Some people still use the text only browser such as lynx which doesn't support images.
  • It helps for SEO, for example search engines can match alt text with search queries.

Width and height

We can also specify the width and height of the image using the width and height attributes. They are integer wihout unit and are represented in pixels.

As soon as the image loads, it will be displayed in the browser. But sometimes the HTML is received but images will take time to load. But when the images are loaded the image is displayed in this case browser has to move the text dowm which is very distracting to the user if they started to read the content. So providing the height and width is very important. Browser knows how much space should be left to fit the image and when recieved the image it displayed it, without moving the text down.

Media assets and licensing

Images and other assets comes with different licenses. We can't just use the random image found over the internet. Let's understand the licensing types

All rights reserved

Sometimes creater of the original work such as songs, books or image release their work under closest copyright licenses. It means only the creator or pubisher have the right to use or distribute it. We need any one of following permission to use it.

  • Written permission by the creator or publisher.
  • Pay the license fee.
  • Limit the use of those work may be considered as fair use

Author are not required to include copyright license term with their work. Copyright automatically exist in original work once it is in tangible medium. So if there is an image in the internet the safest assumption is to consider it under all rights reserved.

Permissive

Some images are released under permissive license such as MIT, BSD or Creative Commons (CC) license, where we do not need a pay or seek the permission to use it, still there are some licensing to fulfill such as

  • Provide a link to the original source of the image and credit its creator.
  • Indicate whether any changes were made to it.
  • Not use the image in any commercial work.
  • Include a copy of the license along with any release that uses the image.
  • Share any derivative works created using the image under the same license as the original.

Sometimes we may hear about copyleft in permissive licenses. Copyleft licenses such as GNU General Public License (GPL) demands that derivates need to be released under same licenses as original.

New project built with the code of a copyleft-licensed which is also know as "fork", will need under the license of same licensed.

Public domain/CC0

Work under the public domain are often refer as "no right reserved", no copyright applies to it and can be used without permission.
