There are security concern about iFrame too.

Security concerns

iFrame are the common target for the hacker to attack. Because of this developer has developed various security mechanism for making iFrame more secure.

The above MDN website is not embeded but give error because the developer has configure it to not allow.

Only embed when necessary

Sometimes it make sense to embed third-project content. A good security rule is You can never be too cautious. If you made it, double-check it anyway. If someone else made it, assume it's dangerous until proven otherwise.

Besides that we should only use the our own content or the content with written permission.

Use HTTPS

HTTPS is the encryption version of HTTP. Website should be served under the HTTPS because

  1. HTTPS reduces the chances that remote content has been tampered in transit.
  2. HTTPS prevents embedded content from accessing content in your parent document, and vice versa.

Many hosting provider offer HTTPS by default.

Always use the sandbox attribute

Content that are not sandbox can use Javascript to use controls the webpage therefore we should use sandbox.

Configure CSP directives

CPS stands for content security policy and provides the sets of HTPP headers designed to improve the security of iFrame. We can use X-Frame-Options header to prevents, this can prevent other websites for embedding the our content.
