    • Debugging the HTML

    When something goes wrong, we can't find where the error is because HTML doesn't have errors.

    HTML and debugging

    HTML is not compiled into different form before the browser parse it. It is interpreted not compiled. HTML code is a lot more permissive than how a programming language is run.

    Permissive code

    When there is error is code, there are generally two type of error, i. e syntax error and logical error.

    Syntax error
    These are usually spelling mistake in the code.

    Logical error
    These are error when the syntax is actually correct but the code doesn't behave as expected, meaninig the code runs incorrectly. These error ususally hard to detect because their is no any error message to direct to error cause.

    HTML doesn't occur the syntax error because browser have built-in rules to state how to interpret the incorrectly written markup code, so there will be something running, even if it is not expected. This can still be a problem. HTML is parsed permissively because when the web was first created, it was decided that allowing people to get their content published was more important than the syntax was absolutely correct.

    Interpreting the error message

    Error messages are ususally helpfull. Lets break download the error of debug-demo.html in debug-demo.html, we get the following errors.

    • End tag li implied, but there were open elements (2 instances): this message indicated the <li> element is opened and not closed. It should be closed.
    • Unclosed element <strong> this message indicated the <strong> element is opened and not closed. It should be closed.
    • End tag <strong> violates nesting rules. this points out the incorrectly nested element.
    • End of file reached when inside an attribute value. Ignoring tag. this indicated that there is attribute value is upto the end of file because it is not closed.
    • End of file seen and there were open elements. this indicated that there are opened element that should be properly closed. An attribute missing a closing quote can result in an open element because the rest of the document is interpreted as the attribute's content.
    • Unclosed element <ul>. this indicated that there is an unclosed <ul> but this is not helpful because there is closing tag. This is because the missing double quote in href attribute of <a> element.
