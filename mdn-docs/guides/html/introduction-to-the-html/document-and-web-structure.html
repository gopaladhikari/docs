    • Document and website structure

    We should define the individual part of the page using we need to plan a basic website structure and the write html to represent the structure.

    Basic sections of a document

    Webpages intended to shared standard structure unless the page is displaying some art such as audio, video or games.

    header:
    Usually a big strip across the top with a big heading, logo, and perhaps a tagline. This usually stays the same from one webpage to another.
    navigation bar:
    Links to the pages within the same website are usually represented by menu, link, button or tabs. So it is very important part of the website, having inconsistent navigation bar will lead to confusion and frustration to user. Many web designer consider navigation bar as the part of the header but it is not required.
    main content:
    This is the main area of the webpage that contains the most of the contents, for example video, or the main story , or the map, news headline etc.
    sidebar:
    In some the webpages sidebar may contain secondary navigation bar. In some the news article where the main content is in main content, author bio can be in sidebar.
    footer:
    It is generally located in the buttom of the webpage and contains copyright notices, contact info, which is generally not critical. The footer is also important for the seo propose.

    HTML for structuring content

    Visual don't tell the whole story. We need to respect semantics and use the right element for the right job.

    To implement such semantic mark up, HTML provides dedicated tags that you can use to represent such sections, for example:

    • header <header>
    • navigation <nav>
    • footer <footer>
    • sidebar <aside>
    • main <main>

    HTML layout elements in more detail

    Main defination of some of the html tags are

    • <header> contains the introductory content. If it is used in the <body> it's a global header, if it is used in <article> or <section> it defines a specific header for that section.
    • <footer> contains the group of end content for a page. Usually it contain copyright notice and contact info.
    • <main> contain the unique content for each page. It should be direct child of <body> element
    • <article> is used for the related content still that make in it's own without the rest of the page (e.g., a single blog post).
    • <section> is used for grouping together different part of page to form a signle page. <section> can be breaked into different <article> and <article> can be breaked into different <section> depending on the content.
    • <aside> is not directly related to main content but can provide extra information indrectly.
    • <nav> contains the main navigation functionality for the page. Secondary links, etc., would not go in the navigation.

    Non-semantic wrappers

    Sometimes we just need a wrappers that don't have semantic meaning, just for css or javascript to use. For this we have the <div> and <span> element

    <span> is inline non-semantic element which should only be used if we don't want to add any semantic meaning to the content. For example

    The King walked drunkenly back to his room at 01:00, the beer doing nothing to aid him as he staggered through the door. [Editor's note: At this point in the play, the lights should be down low].

    Line breaks and horizontal rules

    We can use <br> and <hr> to add break or horizontal line.

    <br> the line break element

    It creates a line break in a paragraph. In can be used in peom, address etc. For example

    There once was a man named O'Dell
    Who loved to write HTML
    But his structure was bad, his semantics were sad
    and his markup didn't read very well.

    <hr> the thematic break element

    It created horizontal rule in the document that denotes a thematic change in the text (such as change in scene or topic ). Visually it looks like a horizontal line. For example

    Ron was backed into a corner by the marauding netherbeasts. Scared, but determined to protect his friends, he raised his wand and prepared to do battle, hoping that his distress call had made it through.

    Meanwhile, Harry was sitting at home, staring at his royalty statement and pondering when the next spin off series would come out, when an enchanted distress letter flew through his window and landed in his lap. He read it hazily and sighed; "better get back to work then", he mused.

    Planning a simple website

    We should plan the structure of the website, what content should put on, what pages are needed etc.

    1. To figure out the common elements in most of the pages such as header, footer etc.
    2. Draw the rough sketch of sructure of the website.
    3. Think about all other non common content of the website.
    4. Sort all these non common into groups, this gives idea what might be similar in different pages.
    5. Now try to sketch a rough sitemap to show the web flow of the website including notes about appearance if required. The homepage will probably be in the center, and link to most if not all of the others; most of the pages in a small site should be available from the main navigation, although there are exceptions.
